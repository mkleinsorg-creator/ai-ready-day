# Tech-Briefing für BMAD · Pfad: n8n-Workflow

> Diese Datei ist **kein User-Prompt**, sondern ein **Pflicht-Kontext für BMAD**.
> Der Master-Prompt übergibt diesen Inhalt beim Start von `/bmad` an die BMAD-Agenten.
> BMAD-Architect und BMAD-Dev richten ihre Empfehlungen nach diesem Briefing.

## Plattform & Constraints

- **n8n Community-Edition** wird genutzt (falkemedia-Instanz)
- **Kein n8n-MCP** verfügbar wegen Community-Edition-Limit
- Daher: Workflows werden als **JSON generiert**, vom User per Drag-and-Drop in n8n importiert
- **8 n8n-Spezialisten-Agenten** sind als Sub-Agenten verfügbar:
  - `n8n-orchestrator` — Hauptsteuerung
  - `n8n-workflow-patterns` — Pattern-Bibliothek
  - `n8n-node-configuration` — Node-Setup
  - `n8n-validation-expert` — Workflow-Validierung
  - `n8n-mcp-tools-expert` — MCP-Tooling (nur nutzen, wenn MCP-fähige Nodes gefragt sind)
  - `n8n-expression-syntax` — Expression-Hilfe
  - `n8n-code-javascript` / `n8n-code-python` — Code-Nodes

## Was BMAD-Architect berücksichtigen muss

- **Klein anfangen:** Trigger + 1 Aktion. Erst zum Laufen bringen, dann erweitern.
- **Nutze `n8n-orchestrator` als ersten Agent** für die Workflow-Struktur. Er fragt nach Trigger, Datenquellen, Zielen.
- **Workflow-Patterns prüfen:** Bevor du eigene Architektur erfindest — `n8n-workflow-patterns` für bewährte Strukturen konsultieren.
- **JSON-Validität:** `n8n-validation-expert` für jeden generierten Workflow durchlaufen lassen, bevor der User importiert.

## Was BMAD-Dev berücksichtigen muss

- **Output ist immer ein JSON-File** in `~/ai-ready-day/projects/<projektname>/workflow.json`
- **Erklär dem User den Import:**
  1. *"Öffne die falkemedia-n8n-Instanz im Browser."*
  2. *"Klick '+ Workflow' → '...' → 'Import from File'."*
  3. *"Wähl die Datei aus dem Projekt-Ordner aus."*
- **Credentials manuell:** Slack-/Asana-/Google-Tokens muss der User selbst in n8n hinterlegen — die wandern NICHT durchs JSON. Anleitung pro Service mitgeben.

## Was BMAD-QA berücksichtigen muss

- **Test in n8n selbst:** "Execute Workflow"-Button oben rechts → Output anschauen
- **Bei Fehlern:** Fehlertext aus n8n kopieren, in Claude einfügen → BMAD-Dev oder `n8n-validation-expert` analysiert und gibt korrigiertes JSON
- **Iteration:** User exportiert geänderten Workflow aus n8n, kopiert JSON in Claude, BMAD updated, User re-importiert.

## Erreichbare Ziel-Systeme (typisch)

- Slack (über falkemedia-Workspace)
- Asana
- Google Sheets / Drive
- HTTP-APIs (z. B. Metricool)
- Eigene Datenbank-Calls

Vor dem Build prüfen: **Hat der User Zugriff/Tokens auf die nötigen Ziel-Systeme?**

## Erfolgs-Definition für diesen Pfad

Am Ende des Tages hat der User:
1. Einen lauffähigen Workflow in n8n
2. Mindestens **einmal manuell ausgelöst** und Ergebnis im Ziel-System gesehen
3. Bonus: Schedule eingestellt, damit der Workflow autonom läuft

Wenn nur 1 + 2 erreicht werden: völlig okay.
