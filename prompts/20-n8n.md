# Pfad: n8n-Workflow bauen

## Tools verfügbar
- **8 n8n-Spezialisten-Agenten** (orchestrator, workflow-patterns, validation-expert, code-javascript, code-python, expression-syntax, mcp-tools-expert, node-configuration)
- **BMAD-Methode** für die Ideation (sollte schon gelaufen sein)

⚠️ **Wichtig:** Wir nutzen die n8n-Community-Edition, daher ist kein direkter MCP-Zugriff verfügbar. Workflows werden als JSON generiert, vom User heruntergeladen und in n8n importiert.

## Vorgehen

### 1. Use-Case aus BMAD übernehmen
Klarer Use-Case: was triggert den Workflow, was sind die Datenquellen, was sind die Zielsysteme?

### 2. n8n-orchestrator aktivieren
Sag dem User:
> *"Ich nutze jetzt unseren n8n-Spezialisten, um den Workflow zu strukturieren."*

Nutze den Agenten `n8n-orchestrator`. Er fragt nach:
- Trigger (Webhook, Schedule, Manual)
- Datenquellen (APIs, Datenbanken, Slack, etc.)
- Verarbeitungsschritte
- Zielsysteme

### 3. Workflow als JSON generieren
Lass den Agenten das fertige n8n-Workflow-JSON erzeugen. Speichere es:

```bash
mkdir -p ~/ai-ready-day/projects/<name>
# JSON in ~/ai-ready-day/projects/<name>/workflow.json speichern
```

### 4. JSON in n8n importieren

Sag dem User Schritt für Schritt:
1. *"Öffne die falkemedia-n8n-Instanz im Browser."*
2. *"Klick auf '+ Workflow' oben rechts."*
3. *"Klick auf das '...' Menü → 'Import from File'."*
4. *"Wähl die Datei `~/ai-ready-day/projects/<name>/workflow.json` aus."*

### 5. Credentials einrichten
Der User muss die Tokens für Slack, Asana, Google etc. einmalig in n8n hinterlegen. n8n führt ihn dabei selbst durch den Prozess.

Falls etwas unklar ist: in n8n den Node anklicken → "Create New Credential" → den OAuth-Flow durchlaufen.

### 6. Mit Test-Trigger testen
n8n hat in jedem Workflow einen "Execute Workflow"-Knopf oben rechts. Damit kannst du den Workflow manuell auslösen.

### 7. Iteration

Wenn etwas nicht passt:
1. In n8n: Workflow exportieren ("Download" als JSON)
2. User kopiert das JSON in den Chat
3. Du analysierst, was schiefläuft
4. Generiere die korrigierte Version
5. User importiert sie wieder

## Wichtige Prinzipien für diesen Pfad

- Vorher klären: Sind die Zielsysteme (Slack, Asana, etc.) erreichbar und hat der User Tokens dafür?
- **Klein anfangen:** Trigger + 1 Aktion zuerst. Erst wenn das läuft, erweitern.
- Bei Fehlern in n8n: User soll den Fehlertext kopieren, Du übersetzt ihn auf Deutsch.
- Erfolg sichtbar machen — z. B. *"Dein Workflow hat gerade automatisch eine Slack-Nachricht geschickt."*
